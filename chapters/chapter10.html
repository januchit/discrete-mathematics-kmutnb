10.1

Graphs: Definitions

นิยามกราฟ, ดีกรี, Handshaking Lemma, กราฟพิเศษ

นิยาม

กราฟ (Graph) $G = (V, E)$ ประกอบด้วยเซตของ จุดยอด (vertices) $V$ และเซตของ ขอบ (edges) $E \subseteq \{\{u,v\} \mid u,v \in V,\, u \neq v\}$

  • ดีกรี $\deg(v)$: จำนวนขอบที่เชื่อมกับจุดยอด $v$
  • Handshaking Lemma: $\displaystyle\sum_{v \in V} \deg(v) = 2|E|$ (ผลรวมดีกรีทุกจุดเท่ากับสองเท่าของจำนวนขอบ)
  • ผลสืบเนื่อง: จำนวนจุดยอดที่มีดีกรีคี่ต้องเป็นเลขคู่เสมอ

กราฟพิเศษที่ควรรู้

  • $K_n$ — Complete graph: ทุกคู่จุดยอดเชื่อมกัน, $|E| = \binom{n}{2}$
  • $K_{m,n}$ — Complete bipartite graph: สองกลุ่ม $m$ และ $n$ จุด, $|E|=mn$
  • $C_n$ — Cycle graph: วงจรที่มี $n$ จุดยอด, ทุกจุดมี $\deg = 2$
ตัวอย่าง 10.1 กราฟ 4 จุดยอด

กราฟ $G$ มีจุดยอด $\{1,2,3,4\}$ และขอบ $\{1\text{-}2,\,1\text{-}3,\,2\text{-}3,\,3\text{-}4\}$

$\deg(1)=2,\;\deg(2)=2,\;\deg(3)=3,\;\deg(4)=1$
ผลรวม $= 2+2+3+1 = 8 = 2 \times 4 = 2|E|$ ✓
10.2

Trails, Paths, and Circuits

Walk, Trail, Path, Euler, Hamiltonian

นิยาม
  • Walk: ลำดับจุดยอดและขอบ $v_0,e_1,v_1,\ldots,v_k$ (ซ้ำได้)
  • Trail: Walk ที่ไม่ซ้ำขอบ
  • Path: Walk ที่ไม่ซ้ำจุดยอด (และขอบ)
  • Circuit: Trail ที่ปิด ($v_0 = v_k$) | Cycle: Path ที่ปิด

ทฤษฎีบท Euler

  • Euler Circuit มีอยู่ก็ต่อเมื่อ กราฟเชื่อมโยง (connected) และ ทุกจุดยอดมีดีกรีคู่
  • Euler Path (ไม่ปิด) มีอยู่ก็ต่อเมื่อ กราฟเชื่อมโยงและมี จุดยอดดีกรีคี่ exactly 2 จุด
  • Hamiltonian Circuit: วงจรที่ผ่านทุกจุดยอดครบถ้วนพอดี (ไม่มีเงื่อนไขง่าย — NP-complete)
ตัวอย่าง 10.2 สะพาน Königsberg

เมือง Königsberg มีเกาะ 4 ส่วน (A,B,C,D) เชื่อมด้วยสะพาน 7 แห่ง ดีกรีของแต่ละโหนด: $\deg(A)=3,\,\deg(B)=3,\,\deg(C)=5,\,\deg(D)=3$

มีจุดยอดดีกรีคี่ถึง 4 จุด (A, B, C, D ล้วนคี่) → ไม่มี Euler Path หรือ Euler Circuit
Euler พิสูจน์ว่าเป็นไปไม่ได้ที่จะเดินข้ามสะพานทุกแห่งครบโดยไม่ซ้ำ
10.3

Matrix Representations

Adjacency Matrix การแทนกราฟด้วยเมทริกซ์

นิยาม

Adjacency Matrix ของกราฟ $G$ ที่มีจุดยอด $v_1,\ldots,v_n$ คือเมทริกซ์ $A$ ขนาด $n \times n$ โดย

$$A_{ij} = \begin{cases} 1 & \text{ถ้า } \{v_i, v_j\} \in E \\ 0 & \text{อื่น ๆ} \end{cases}$$

กราฟไม่มีทิศทาง: $A$ เป็น symmetric matrix ($A = A^T$)

10.4

Graph Isomorphisms

ไอโซมอร์ฟิซึม — กราฟ "เหมือนกัน" ในเชิงโครงสร้าง

นิยาม

กราฟ $G_1=(V_1,E_1)$ และ $G_2=(V_2,E_2)$ ไอโซมอร์ฟิค กัน ($G_1 \cong G_2$) หากมี bijection $f: V_1 \to V_2$ ซึ่ง $\{u,v\} \in E_1 \iff \{f(u),f(v)\} \in E_2$

Invariants (ต้องเท่ากันทั้งคู่): $|V|$, $|E|$, degree sequence (เรียงจากน้อยไปมาก)

10.5

Trees

ต้นไม้ — กราฟเชื่อมโยงที่ไม่มีวงจร

นิยาม

ต้นไม้ (Tree) คือกราฟที่ เชื่อมโยง (connected) และ ไม่มี cycle

ทฤษฎีบท: ต้นไม้ที่มี $n$ จุดยอดมีขอบพอดี $n-1$ ขอบ

Leaf: จุดยอดที่มี $\deg = 1$ ต้นไม้ที่มี $n \geq 2$ จุดต้องมี leaf อย่างน้อย 2 จุด

10.6

Rooted Trees

ต้นไม้มีราก, Binary Tree, Expression Tree

นิยาม

Rooted Tree: ต้นไม้ที่กำหนดจุดยอดหนึ่งเป็น ราก (root)

  • Parent: จุดยอดที่อยู่บน | Child: จุดยอดที่อยู่ล่างทันที
  • Leaf: จุดยอดที่ไม่มี child | Height: ความยาวเส้นทางยาวที่สุดจาก root ถึง leaf
  • Binary Tree: Rooted Tree ที่แต่ละจุดมี child ได้ไม่เกิน 2 จุด (left/right)
ตัวอย่าง 10.6 Expression Tree สำหรับ $(a+b) \cdot c$
Root = ×, height = 2, leaves = {a, b, c}
การคำนวณ Postorder: $a,\,b,\,+,\,c,\,\times$ (Reverse Polish Notation)
10.7

Spanning Trees & Shortest Paths

Kruskal's Algorithm, Dijkstra's Algorithm

นิยาม

Spanning Tree ของกราฟ $G$ คือ subgraph ที่เป็น tree และมีจุดยอดครบทุกจุดของ $G$

Minimum Spanning Tree (MST): Spanning Tree ที่น้ำหนักรวมของขอบน้อยที่สุด (สำหรับ weighted graph)

Kruskal's Algorithm

เรียงขอบตามน้ำหนัก จากน้อยไปมาก เพิ่มขอบเข้า MST ทีละขอบหากไม่สร้าง cycle
ตัวอย่าง 10.7a Kruskal บน weighted graph 5 จุด

ขอบทั้งหมด (เรียงตามน้ำหนัก): A-B:1, C-D:2, A-C:3, B-C:4, B-D:5, D-E:6

ขั้นขอบที่พิจารณาน้ำหนักเพิ่ม MST?เหตุผล
1A–B1✅ ใช่ไม่สร้าง cycle
2C–D2✅ ใช่ไม่สร้าง cycle
3A–C3✅ ใช่ไม่สร้าง cycle
4B–C4❌ ไม่A-B-C ทำให้เกิด cycle
5D–E6✅ ใช่ครบ 4 ขอบ = MST สมบูรณ์
MST: {A-B, C-D, A-C, D-E} น้ำหนักรวม $= 1+2+3+6 = 12$

Dijkstra's Algorithm

หาเส้นทางสั้นที่สุดจาก source ไปยังจุดยอดอื่น ทำงานบน weighted graph ที่น้ำหนักไม่ติดลบ
  1. กำหนดระยะ $d(s)=0$ สำหรับ source $s$ และ $d(v)=\infty$ ทุกจุดอื่น
  2. เลือกจุดยอด $u$ ที่ยังไม่ได้ visit และมี $d(u)$ น้อยที่สุด
  3. สำหรับทุก neighbor $v$ ของ $u$: ถ้า $d(u)+w(u,v) < d(v)$ แล้ว update $d(v)$
  4. ทำเครื่องหมาย $u$ ว่า visited แล้วกลับขั้น 2
  5. หยุดเมื่อ visited ทุกจุดหรือถึงจุดหมาย

สรุปบทที่ 10

กราฟพื้นฐาน

  • $G=(V,E)$, $\deg(v)$, Handshaking Lemma
  • $K_n$, $K_{m,n}$, $C_n$
  • Adjacency Matrix: $n\times n$, symmetric
  • Isomorphism: $|V|$, $|E|$, degree sequence

เส้นทาง

  • Walk → Trail → Path
  • Euler circuit: connected + all even deg
  • Euler path: exactly 2 odd-deg vertices
  • Hamiltonian: NP-complete (ไม่มีเงื่อนไขง่าย)

ต้นไม้

  • Tree: connected + acyclic, $|E|=n-1$
  • Rooted tree: root, parent, child, leaf, height
  • Binary tree: child ≤ 2 ต่อจุด
  • MST: Kruskal (greedy ตามน้ำหนักขอบ)
  • Shortest path: Dijkstra (non-negative weights)
