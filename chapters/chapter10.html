10.1

Graphs: Definitions and Basic Properties

นิยามกราฟ, degree, handshaking lemma, กราฟพิเศษ

นิยามกราฟ

นิยาม: Graph (กราฟ)

กราฟ $G = (V, E)$ ประกอบด้วย $V$ = เซต จุดยอด (vertices) ที่ไม่ว่าง และ $E$ = เซต เส้นเชื่อม (edges)

  • Simple graph: ไม่มี self-loop และไม่มี parallel edges
  • Multigraph: อนุญาต parallel edges ระหว่างคู่จุดยอดเดียวกัน
  • Pseudograph: อนุญาตทั้ง self-loop และ parallel edges
  • Digraph: เส้นเชื่อมมีทิศทาง $(u \to v)$
ตัวอย่าง 10.1.1 Simple Graph

$V = \{1,2,3,4\}$, $E = \{(1,2),(1,3),(2,3),(2,4),(3,4)\}$, $|V|=4$, $|E|=5$

$\deg(1)=2,\ \deg(2)=3,\ \deg(3)=3,\ \deg(4)=2$

Degree และ Handshaking Lemma

นิยาม: Degree ของจุดยอด

$\deg(v)$ = จำนวนเส้นเชื่อมที่ต่อกับ $v$ (self-loop นับ 2)

Digraph: in-degree $\deg^-(v)$ = เส้นเข้า, out-degree $\deg^+(v)$ = เส้นออก

ทฤษฎีบท: Handshaking Lemma

$$\sum_{v \in V} \deg(v) = 2|E|$$

บทแทรก: จำนวนจุดยอดที่มี degree คี่ต้องเป็นเลขคู่เสมอ

🔍 พิสูจน์ Handshaking Lemma
สังเกต

แต่ละเส้นเชื่อม $(u,v)$ นับเพิ่ม $\deg(u)$ หนึ่ง และ $\deg(v)$ หนึ่ง → 2 ต่อ 1 เส้น

รวม

$\sum_{v \in V} \deg(v) = \sum_{e \in E} 2 = 2|E|$

บทแทรก

ถ้ามีจุดยอด degree คี่ $k$ ตัว ผลรวม $= 2|E|$ (คู่) ดังนั้น $k$ ต้องคู่

∎

กราฟพิเศษ

ประเภทกราฟพิเศษสำคัญ

  • $K_n$ (Complete graph): ทุกคู่จุดยอดมีเส้นเชื่อม — $|E|=\binom{n}{2}$, ทุก $\deg=n-1$
  • $K_{m,n}$ (Complete bipartite): $V=A\cup B$, $|A|=m$, $|B|=n$, เชื่อมทุกคู่ $(a,b)$ — $|E|=mn$
  • $C_n$ (Cycle): $n$ จุดเรียงวงกลม, $|E|=n$, ทุก $\deg=2$
  • $P_n$ (Path): $n$ จุดเรียงเส้นตรง, $|E|=n-1$
  • $W_n$ (Wheel): $C_{n-1}$ + hub ตรงกลาง, $|E|=2(n-1)$
ตัวอย่าง 10.1.2 $K_4$

$K_4$: 4 จุดยอด, $|E|=\binom{4}{2}=6$ เส้น, ทุก $\deg=3$

ตรวจสอบ Handshaking: $\sum\deg = 3+3+3+3 = 12 = 2\times6 = 2|E|$ ✓

10.2

Trails, Paths, and Circuits

เส้นทาง วิถี วงจร และทฤษฎีบทออยเลอร์

Walk, Trail, Path, Circuit

นิยาม: ประเภทเส้นทาง

  • Walk: ลำดับ $v_0,e_1,v_1,\ldots,e_k,v_k$ — ซ้ำได้ทั้ง vertices และ edges
  • Trail: Walk ที่ไม่ซ้ำ edges
  • Path: Walk ที่ไม่ซ้ำ vertices
  • Circuit: Trail ปิด ($v_0=v_k$)
  • Cycle: Path ปิด ($v_0=v_k$, ไม่ซ้ำ vertex กลาง)

ลำดับ: Path $\subset$ Trail $\subset$ Walk

นิยาม: Connected Graph

กราฟ $G$ เป็น connected ถ้าทุกคู่จุดยอด $u,v$ มี path เชื่อมกัน

ตัวอย่าง 10.2.1 แยกประเภทเส้นทาง

บนกราฟ $V=\{1,2,3,4\}$, $E=\{(1,2),(1,3),(2,3),(2,4),(3,4)\}$:

  • $1\to2\to3\to1\to2$ — Walk (ซ้ำ vertex 1,2 และ edge $(1,2)$)
  • $1\to2\to4\to3\to1$ — Circuit (trail ปิด, ไม่ซ้ำ edge)
  • $1\to2\to4$ — Path (ไม่ซ้ำ vertex หรือ edge)

Euler Circuit และ Euler Path

นิยาม: Euler Circuit / Path

  • Euler circuit: Circuit ที่ผ่านทุก edge ตรงหนึ่งครั้ง
  • Euler path: Trail ที่ผ่านทุก edge ตรงหนึ่งครั้ง (ไม่ต้องปิด)

ทฤษฎีบทออยเลอร์ (1736)

กราฟ $G$ connected มี Euler circuit ⇔ จุดยอดทุกตัวมี degree คู่

กราฟ $G$ connected มี Euler path (ไม่ใช่ circuit) ⇔ มีจุดยอด degree คี่ตรง 2 ตัว

ตัวอย่าง 10.2.2 สะพาน Königsberg (1736)

4 ส่วนของเมือง เชื่อมด้วย 7 สะพาน: $\deg(A)=3,\ \deg(B)=3,\ \deg(C)=5,\ \deg(D)=3$ (คี่ทั้ง 4)

มีจุดยอด degree คี่ถึง 4 ตัว → ไม่มีทั้ง Euler circuit และ Euler path! — นี่คือจุดกำเนิดทฤษฎีกราฟ
ตัวอย่าง 10.2.3 ตรวจสอบ Euler Circuit

กราฟ $G$: degree sequence $= (2,4,4,2,4)$ ทุกตัวเป็นเลขคู่, connected → มี Euler circuit ✓

Hierholzer's algorithm: หา Euler circuit ใน $O(E)$ — เริ่มจาก vertex ใดๆ เดิน trail จนกลับมา แล้วต่อ sub-circuit ที่ยังเหลือ

Hamiltonian Circuit

นิยาม: Hamiltonian Circuit / Path

  • Hamiltonian circuit: Cycle ที่ผ่านทุก vertex ตรงหนึ่งครั้ง
  • Hamiltonian path: Path ที่ผ่านทุก vertex ตรงหนึ่งครั้ง
Hamiltonian vs Euler:
  • Euler = ทุก edge → ตรวจ $O(V+E)$ ด้วย degree condition
  • Hamiltonian = ทุก vertex → ปัญหา NP-complete, ไม่มีทฤษฎีบทง่ายๆ
  • Traveling Salesman Problem (TSP) = หา minimum Hamiltonian circuit ใน weighted graph
10.3

Matrix Representations of Graphs

การแทนกราฟด้วย Adjacency Matrix

Adjacency Matrix

นิยาม: Adjacency Matrix $A$

Matrix ขนาด $n\times n$ โดย $A[i][j]=1$ ถ้า $(v_i,v_j)\in E$ และ $0$ อื่นๆ

กราฟไม่มีทิศทาง: $A$ เป็น symmetric ($A=A^T$), ผลรวมแถว $i$ $= \deg(v_i)$

$G$: $V=\{1,2,3,4\}$, $E=\{(1,2),(1,3),(2,3),(2,4),(3,4)\}$

ผลรวมแถว: $2,3,3,2$ = degree sequence ✓

กำลังของ Adjacency Matrix

ทฤษฎีบท: $(A^k)[i][j]$ = จำนวน Walk ยาว $k$

$(A^k)[i][j]$ = จำนวน walk ความยาว $k$ จาก $v_i$ ถึง $v_j$

ตัวอย่าง 10.3.2 นับ Walk ยาว 2

$(A^2)[1][4]$ = จำนวน walk ยาว 2 จาก 1 ถึง 4 = walks $1\to2\to4$ และ $1\to3\to4$ = 2

คำนวณ: $(A^2)[1][4] = A[1][2]\cdot A[2][4] + A[1][3]\cdot A[3][4] = 1\cdot1 + 1\cdot1 = 2$ ✓

การประยุกต์ใช้ Adjacency Matrix

  • ตรวจ connectivity ด้วย BFS/DFS ผ่าน $A$
  • นับ walks ด้วย matrix multiplication
  • PageRank ของ Google ใช้ adjacency matrix ของ web graph
10.4

Isomorphisms of Graphs

กราฟที่มีโครงสร้างเหมือนกัน

นิยาม Graph Isomorphism

นิยาม: Graph Isomorphism

$G=(V,E)$ และ $H=(W,F)$ เป็น isomorphic ($G\cong H$) ถ้ามี bijection $f:V\to W$ ที่

$$(u,v)\in E \iff (f(u),f(v))\in F$$
ตัวอย่าง 10.4.1 สร้าง Isomorphism

$G_1$: $C_4 = \{1,2,3,4\}$, $E_1=\{(1,2),(2,3),(3,4),(4,1)\}$

$G_2$: $\{a,b,c,d\}$, $E_2=\{(a,c),(c,b),(b,d),(d,a)\}$

$f: 1\mapsto a, 2\mapsto c, 3\mapsto b, 4\mapsto d$ ตรวจ: $(1,2)\Rightarrow(a,c)$ ✓ ทุก edge ✓ → $G_1\cong G_2$

Graph Invariants

$|V|$ และ $|E|$

จำนวน vertices และ edges ต้องเท่ากัน

Degree Sequence

ลำดับ degree เรียงน้อย→มากต้องเหมือนกัน

Connectedness

จำนวน connected components ต้องเท่ากัน

Girth

ความยาว cycle สั้นสุดต้องเท่ากัน

จำนวน Cycles

ต้องเท่ากันสำหรับทุกความยาว $k$

Bipartiteness

bipartite/non-bipartite ต้องตรงกัน

ตัวอย่าง 10.4.2 พิสูจน์ว่าไม่ Isomorphic

$G_1$: degree sequence $(1,1,3,3)$; $G_2$: degree sequence $(2,2,2,2)$

Degree sequence ต่างกัน → $G_1\not\cong G_2$ ∎
ตัวอย่าง 10.4.3 Invariant เหมือนแต่ยังต้องตรวจต่อ

$G_1=C_6$ (วงกลม 6 จุด), $G_2=C_3\cup C_3$ (สองสามเหลี่ยม)

ทั้งคู่: $|V|=6$, $|E|=6$, degree sequence $(2,2,2,2,2,2)$ — เหมือนกันทุกอย่าง

แต่: $G_1$ connected, $G_2$ ไม่ connected → $G_1\not\cong G_2$ ∎

10.5

Trees

ต้นไม้ — connected acyclic graph

นิยามต้นไม้

นิยาม: Tree และ Forest

  • Tree: กราฟที่ connected และ acyclic
  • Forest: กราฟ acyclic (อาจไม่ connected)
  • Leaf: vertex ที่ $\deg=1$

ทฤษฎีบทสำคัญของ Tree

ทฤษฎีบท: สมบัติสมมูลของ Tree

สำหรับ graph $G$ ที่มี $n$ vertices สิ่งต่อไปนี้สมมูลกัน:

  1. $G$ เป็น tree (connected + acyclic)
  2. $G$ connected และมี $n-1$ edges
  3. $G$ acyclic และมี $n-1$ edges
  4. ทุกคู่ vertices มี unique path เชื่อมกัน
ตัวอย่าง 10.5.1 ตรวจสอบว่าเป็น Tree

$G$: $V=\{1,2,3,4,5\}$, $E=\{(1,2),(1,3),(3,4),(3,5)\}$

$|V|=5$, $|E|=4=n-1$ ✓, Connected ✓ → เป็น Tree

ตัวอย่าง 10.5.2 File System
  • / (root)
    • home/
      • user/
        • documents/
        • downloads/
    • usr/
    • etc/

$n=7$ nodes, $|E|=6=n-1$ ✓

10.6

Rooted Trees

ต้นไม้มีราก, Binary Tree, Traversal

ต้นไม้มีราก

นิยาม: Rooted Tree

Tree ที่กำหนดจุดยอดพิเศษหนึ่งเป็น root

  • Parent/Child: vertex ที่อยู่เหนือ/ใต้โดยตรงบน path จาก root
  • Leaf: vertex ไม่มี child  |  Internal node: มี child ≥1 ตัว
  • Level(v): ความยาว path จาก root ถึง $v$
  • Height: level สูงสุดของ tree

Binary Tree

นิยาม: Binary Tree

  • Binary tree: แต่ละ node มี child ≤2 (left, right)
  • Full binary tree: ทุก internal node มี child ตรง 2 ตัว
  • Complete binary tree: ทุก level เต็ม ยกเว้น level สุดท้ายที่เรียงชิดซ้าย

ทฤษฎีบท: Full Binary Tree

Full binary tree ที่มี $i$ internal nodes มี $i+1$ leaves และ $2i+1$ nodes รวม

Binary tree สูง $h$: leaves $\le 2^h$, nodes $\le 2^{h+1}-1$

Tree Traversal

นิยาม: Traversal 3 แบบ

  • Preorder: Root → Left → Right
  • Inorder: Left → Root → Right (ได้ sorted order สำหรับ BST)
  • Postorder: Left → Right → Root
ตัวอย่าง 10.6.1 Expression Tree สำหรับ $(a+b)\times(c-d)$
  • Preorder (prefix): $\times\ +\ a\ b\ -\ c\ d$
  • Inorder (infix): $a+b\times c-d$
  • Postorder (RPN): $a\ b+\ c\ d-\ \times$
ตัวอย่าง 10.6.2 Binary Search Tree (BST)

BST: ทุก node ใน left subtree $<$ root $<$ ทุก node ใน right subtree

ใส่ $\{5,3,7,1,4,6,8\}$: Inorder traversal ได้ $1,3,4,5,6,7,8$ (sorted) ✓

Search/Insert/Delete ใน balanced BST: $O(\log n)$

10.7

Spanning Trees and Shortest Paths

MST, Kruskal's Algorithm, Dijkstra's Algorithm

Spanning Tree และ MST

นิยาม: Spanning Tree และ MST

  • Spanning tree: subgraph ที่เป็น tree และมีทุก vertex ($n$ vertices, $n-1$ edges)
  • Minimum spanning tree (MST): spanning tree ที่ผลรวมน้ำหนัก edges น้อยที่สุด

MST ใช้ทำอะไร?

  • วางสายโทรศัพท์/ไฟฟ้าเชื่อมทุกเมืองด้วยต้นทุนน้อยที่สุด
  • Cluster analysis ใน machine learning
  • Network routing protocols (OSPF, STP)

Kruskal's Algorithm

อัลกอริทึม Kruskal (1956) — Greedy MST

  1. เรียง edges ทุกเส้นตามน้ำหนักจากน้อยไปมาก
  2. วน: เพิ่ม edge เบาที่สุดที่ยังไม่ก่อ cycleเข้า MST
  3. หยุดเมื่อได้ $n-1$ edges

ใช้ Union-Find ตรวจ cycle — $O(E\log E)$

ตัวอย่าง 10.7.1 Kruskal บนกราฟ 5 จุดยอด

$V=\{A,B,C,D,E\}$, edges เรียงตามน้ำหนักแล้ว:

ขั้นEdgeน้ำหนักก่อ Cycle?ผล
1(A,B)1ไม่✓ เพิ่มเข้า MST
2(B,C)2ไม่✓ เพิ่มเข้า MST
3(A,C)3ใช่ (A–B–C–A)✗ ข้าม
4(B,D)4ไม่✓ เพิ่มเข้า MST
5(C,D)5ใช่ (B–C–D–B)✗ ข้าม
6(C,E)6ไม่✓ เพิ่ม (ครบ 4 edges)

MST: $(A,B),(B,C),(B,D),(C,E)$, น้ำหนักรวม $=1+2+4+6=13$

Dijkstra's Shortest Path Algorithm

อัลกอริทึม Dijkstra (1959)

  1. $d[s]=0$, $d[v]=\infty$ สำหรับ $v\ne s$
  2. เลือก unvisited vertex $u$ ที่ $d[u]$ น้อยที่สุด
  3. สำหรับ neighbor $v$: ถ้า $d[u]+w(u,v)<d[v]$ → อัปเดต $d[v]$ (Relaxation)
  4. ทำเครื่องหมาย $u$ visited แล้ววนซ้ำ

ความซับซ้อน: $O((V+E)\log V)$ ด้วย min-heap  |  ข้อจำกัด: ต้องการ edge weight ≥0
