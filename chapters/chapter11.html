11.1

Real-Valued Functions of a Real Variable and Their Graphs

ฟังก์ชันค่าจริง, กราฟ, Floor, Ceiling, และการเปรียบเทียบอัตราการเติบโต

Floor และ Ceiling Functions

นิยาม: Floor Function (ฟังก์ชันพื้น)

สำหรับจำนวนจริง $x$ นิยาม Floor ของ $x$ คือ

$$\lfloor x \rfloor = \text{จำนวนเต็มที่มากที่สุดที่ไม่เกิน } x$$

นั่นคือ $\lfloor x \rfloor = n$ ก็ต่อเมื่อ $n \leq x < n+1$ โดยที่ $n \in \mathbb{Z}$

นิยาม: Ceiling Function (ฟังก์ชันเพดาน)

สำหรับจำนวนจริง $x$ นิยาม Ceiling ของ $x$ คือ

$$\lceil x \rceil = \text{จำนวนเต็มที่น้อยที่สุดที่ไม่น้อยกว่า } x$$

นั่นคือ $\lceil x \rceil = n$ ก็ต่อเมื่อ $n - 1 < x \leq n$ โดยที่ $n \in \mathbb{Z}$

ตัวอย่าง 11.1.1 คำนวณ Floor และ Ceiling

คำนวณค่า floor และ ceiling ของจำนวนต่อไปนี้

$x$$\lfloor x \rfloor$$\lceil x \rceil$หมายเหตุ
$3.7$$3$$4$$3 \leq 3.7 < 4$
$-2.1$$-3$$-2$$-3 \leq -2.1 < -2$
$5$$5$$5$จำนวนเต็ม: floor = ceil = ตัวเอง
$-0.5$$-1$$0$$-1 \leq -0.5 < 0$
$\pi \approx 3.14159$$3$$4$$3 \leq \pi < 4$

ข้อสังเกต: ถ้า $x$ เป็นจำนวนเต็ม $\lfloor x \rfloor = \lceil x \rceil = x$; ถ้าไม่ใช่ $\lfloor x \rfloor + 1 = \lceil x \rceil$

ตัวอย่าง 11.1.2 Floor ในการเขียนโปรแกรม

ในภาษา Python/C: n // 2 คือ $\lfloor n/2 \rfloor$ และ math.ceil(n/2) คือ $\lceil n/2 \rceil$

สำหรับ $n = 7$: $\lfloor 7/2 \rfloor = 3$ (ผลหารจำนวนเต็ม) และ $\lceil 7/2 \rceil = 4$

ใช้ใน Binary Search: midpoint $= \lfloor (lo + hi) / 2 \rfloor$

ใช้ใน Merge Sort: แบ่ง array ขนาด $n$ ออกเป็น $\lfloor n/2 \rfloor$ และ $\lceil n/2 \rceil$

สมบัติสำคัญของ Floor และ Ceiling

  • $\lfloor x \rfloor \leq x < \lfloor x \rfloor + 1$
  • $\lceil x \rceil - 1 < x \leq \lceil x \rceil$
  • $\lfloor -x \rfloor = -\lceil x \rceil$ และ $\lceil -x \rceil = -\lfloor x \rfloor$
  • $\lfloor x + n \rfloor = \lfloor x \rfloor + n$ สำหรับ $n \in \mathbb{Z}$
  • $\lfloor n/2 \rfloor + \lceil n/2 \rceil = n$ สำหรับ $n \in \mathbb{Z}$
ตัวอย่าง 11.1.3 พิสูจน์สมบัติ $\lfloor n/2 \rfloor + \lceil n/2 \rceil = n$

กรณี $n$ คู่: $n = 2k$ ดังนั้น $\lfloor 2k/2 \rfloor + \lceil 2k/2 \rceil = k + k = 2k = n$ ✓

กรณี $n$ คี่: $n = 2k+1$ ดังนั้น $\lfloor (2k+1)/2 \rfloor + \lceil (2k+1)/2 \rceil = k + (k+1) = 2k+1 = n$ ✓

ประเภทของฟังก์ชันและกราฟ

ในการวิเคราะห์อัลกอริทึม เราสนใจฟังก์ชันหลักหลายประเภทที่แสดงถึงจำนวน "ขั้นตอน" ที่อัลกอริทึมใช้เมื่อ input มีขนาด $n$

นิยาม

ฟังก์ชัน Polynomial: $f(n) = a_k n^k + a_{k-1} n^{k-1} + \cdots + a_1 n + a_0$ โดย $a_k \neq 0$

ฟังก์ชัน Exponential: $f(n) = b^n$ โดย $b > 1$ เช่น $2^n$, $3^n$, $10^n$

ฟังก์ชัน Logarithmic: $f(n) = \log_b n$ โดย $b > 1, b \neq 1$ เช่น $\log_2 n$, $\ln n$

ฟังก์ชัน Factorial: $f(n) = n! = 1 \cdot 2 \cdot 3 \cdots n$

11.2

O-, Ω-, and Θ-Notations

สัญลักษณ์ Big-O, Big-Omega, Big-Theta — นิยามอย่างเป็นทางการและการพิสูจน์

O
Big-O (Upper Bound)
$f(n) = O(g(n))$: $f$ โตไม่เร็วกว่า $g$ (asymptotically)
Ω
Big-Ω (Lower Bound)
$f(n) = \Omega(g(n))$: $f$ โตไม่ช้ากว่า $g$ (asymptotically)
Θ
Big-Θ (Tight Bound)
$f(n) = \Theta(g(n))$: $f$ โตเร็วเท่ากับ $g$ (asymptotically)

Big-O Notation (สัญลักษณ์โอใหญ่)

นิยาม: Big-O

ให้ $f, g: \mathbb{Z}^+ \to \mathbb{R}^+$ เป็นฟังก์ชัน เราบอกว่า $f(n)$ คือ $O(g(n))$ (อ่านว่า "$f$ คือ Big-O ของ $g$") ก็ต่อเมื่อ

$$\exists\, c > 0,\; \exists\, n_0 \in \mathbb{Z}^+ \text{ such that } f(n) \leq c \cdot g(n) \text{ for all } n \geq n_0$$

นั่นคือ $f(n)$ ถูก "ขอบเขตบน" โดย $c \cdot g(n)$ เมื่อ $n$ ใหญ่พอ

ตัวอย่าง 11.2.1 พิสูจน์ว่า $5n^2 + 3n + 1 = O(n^2)$
การพิสูจน์
กำหนด

ต้องการหา $c > 0$ และ $n_0$ ที่ทำให้ $5n^2 + 3n + 1 \leq c \cdot n^2$ สำหรับ $n \geq n_0$

หา Bound

สำหรับ $n \geq 1$: $3n \leq 3n^2$ และ $1 \leq n^2$

ดังนั้น $5n^2 + 3n + 1 \leq 5n^2 + 3n^2 + n^2 = 9n^2$

สรุป

เลือก $c = 9$ และ $n_0 = 1$: สำหรับทุก $n \geq 1$ มี $5n^2 + 3n + 1 \leq 9n^2$

ดังนั้น $5n^2 + 3n + 1 = O(n^2)$ ∎

ตัวอย่าง 11.2.2 พิสูจน์ว่า $n^2 \neq O(n)$

สมมติ (เพื่อหา contradiction) ว่า $n^2 = O(n)$ หมายความว่ามี $c > 0$ และ $n_0$ ที่ $n^2 \leq cn$ สำหรับ $n \geq n_0$

นั่นคือ $n \leq c$ สำหรับ $n \geq n_0$ ซึ่งขัดแย้งกับ $n$ สามารถใหญ่ได้ไม่จำกัด

ดังนั้น $n^2 \neq O(n)$ ∎

ตัวอย่าง 11.2.3 พิสูจน์ว่า $100n + \log n = O(n)$

สำหรับ $n \geq 2$: $\log_2 n \leq n$ (เห็นได้จากกราฟ)

ดังนั้น $100n + \log_2 n \leq 100n + n = 101n$

เลือก $c = 101$ และ $n_0 = 2$: จึงได้ $100n + \log n = O(n)$ ∎

ตัวอย่าง 11.2.4 แสดงว่า $\log_2 n = O(\sqrt{n})$

ใช้ผลจากตัวอย่าง 11.1.5: $\log_2 n \leq \sqrt{n}$ สำหรับ $n \geq 4$

เลือก $c = 1$ และ $n_0 = 4$: จึงได้ $\log_2 n = O(\sqrt{n})$ ∎

Big-Ω Notation (สัญลักษณ์โอเมก้าใหญ่)

นิยาม: Big-Ω

เราบอกว่า $f(n)$ คือ $\Omega(g(n))$ ก็ต่อเมื่อ

$$\exists\, c > 0,\; \exists\, n_0 \in \mathbb{Z}^+ \text{ such that } f(n) \geq c \cdot g(n) \text{ for all } n \geq n_0$$

กล่าวคือ $f(n) = \Omega(g(n)) \iff g(n) = O(f(n))$

ตัวอย่าง 11.2.5 พิสูจน์ว่า $3n^2 + 2n = \Omega(n^2)$

ต้องหา $c > 0$ และ $n_0$ ที่ $3n^2 + 2n \geq c \cdot n^2$ สำหรับ $n \geq n_0$

เนื่องจาก $3n^2 + 2n \geq 3n^2$ สำหรับ $n \geq 0$

เลือก $c = 3$ และ $n_0 = 1$: จึงได้ $3n^2 + 2n = \Omega(n^2)$ ∎

Big-Θ Notation (สัญลักษณ์ทีตาใหญ่)

นิยาม: Big-Θ

เราบอกว่า $f(n)$ คือ $\Theta(g(n))$ ก็ต่อเมื่อ $f(n) = O(g(n))$ และ $f(n) = \Omega(g(n))$

เทียบเท่า: $\exists\, c_1, c_2 > 0,\; n_0$ ที่ $c_1 \cdot g(n) \leq f(n) \leq c_2 \cdot g(n)$ สำหรับทุก $n \geq n_0$

ตัวอย่าง 11.2.6 พิสูจน์ว่า $3n^2 + 2n = \Theta(n^2)$
พิสูจน์ทั้ง Upper และ Lower Bound
Upper Bound: $3n^2 + 2n = O(n^2)$

สำหรับ $n \geq 1$: $2n \leq 2n^2$ ดังนั้น $3n^2 + 2n \leq 5n^2$

เลือก $c_2 = 5$, $n_0 = 1$

Lower Bound: $3n^2 + 2n = \Omega(n^2)$

$3n^2 + 2n \geq 3n^2$ สำหรับทุก $n \geq 0$

เลือก $c_1 = 3$, $n_0 = 1$

สรุป

มี $3n^2 \leq 3n^2 + 2n \leq 5n^2$ สำหรับ $n \geq 1$

ดังนั้น $3n^2 + 2n = \Theta(n^2)$ ∎

ตัวอย่าง 11.2.7 Polynomial ทุกตัวมี Theta

ให้ $p(n) = a_k n^k + a_{k-1} n^{k-1} + \cdots + a_0$ โดย $a_k > 0$ แล้ว $p(n) = \Theta(n^k)$

Upper bound: $|p(n)| \leq (|a_k| + |a_{k-1}| + \cdots + |a_0|) n^k = c_2 n^k$ สำหรับ $n \geq 1$

Lower bound: สำหรับ $n$ ใหญ่พอ เทอม $a_k n^k$ ครองทั้งหมด: $p(n) \geq \frac{a_k}{2} n^k$

ดังนั้น $p(n) = \Theta(n^k)$ ∎

สมบัติสำคัญของ O, Ω, Θ

ทฤษฎีบท: Transitivity (สกรรมภาพ)

ถ้า $f = O(g)$ และ $g = O(h)$ แล้ว $f = O(h)$ (เช่นเดียวกันสำหรับ Ω และ Θ)

ทฤษฎีบท: Sum Rule (กฎผลรวม)

ถ้า $f_1 = O(g_1)$ และ $f_2 = O(g_2)$ แล้ว $f_1 + f_2 = O(\max(g_1, g_2))$

ตัวอย่าง: $O(n^2) + O(n) = O(n^2)$ (เทอมที่โตเร็วกว่าครองทั้งหมด)

ทฤษฎีบท: Product Rule (กฎผลคูณ)

ถ้า $f_1 = O(g_1)$ และ $f_2 = O(g_2)$ แล้ว $f_1 \cdot f_2 = O(g_1 \cdot g_2)$

ตัวอย่าง: Loop ซ้อน: $O(n) \times O(n) = O(n^2)$

11.3

Application: Analysis of Algorithm Efficiency I

การวิเคราะห์ประสิทธิภาพส่วนที่ 1 — การนับ Basic Operations, Loop Analysis

หลักการวิเคราะห์อัลกอริทึม

  • Basic Operation: คือ operation ที่ใช้เวลา $O(1)$ เช่น การเปรียบเทียบ, การบวก, การ assign ค่า
  • Worst-case: จำนวน ops สูงสุดที่เป็นไปได้สำหรับ input ขนาด $n$
  • Best-case: จำนวน ops น้อยที่สุดที่เป็นไปได้
  • Average-case: ค่าเฉลี่ยของ ops เหนือ distribution ของ input ทั้งหมด

การวิเคราะห์ Loop

ตัวอย่าง 11.3.1 นับ Basic Operations ใน Single Loop
// หาผลรวม array sum = 0 // 1 op for i = 1 to n: // loop n ครั้ง sum = sum + a[i] // 1 op ต่อรอบ (บวก + assign) return sum // 1 op

การนับ: 1 (init) + $n$ (loop body) + 1 (return) $= n + 2$ ops

Complexity: $T(n) = n + 2 = O(n)$ — Linear

ตัวอย่าง 11.3.2 นับ Operations ใน Nested Loop
for i = 1 to n: for j = 1 to n: A[i][j] = i * j // 1 op

การนับ: outer loop รัน $n$ ครั้ง, inner loop รัน $n$ ครั้งต่อหนึ่งรอบ outer

รวมทั้งหมด: $n \times n = n^2$ ops

Complexity: $T(n) = n^2 = O(n^2)$ — Quadratic

ตัวอย่าง 11.3.3 Loop ที่มีจำนวนรอบไม่เท่ากัน
for i = 1 to n: for j = 1 to i: // j วิ่งจาก 1 ถึง i count++

การนับ: เมื่อ $i=1$: $j$ รัน 1 รอบ; $i=2$: 2 รอบ; $\ldots$; $i=n$: $n$ รอบ

รวม: $1 + 2 + 3 + \cdots + n = \dfrac{n(n+1)}{2}$ ops

Complexity: $T(n) = \dfrac{n(n+1)}{2} = O(n^2)$ ถึงแม้จะดีกว่า nested loop เต็มประมาณ 2 เท่า

ตัวอย่าง 11.3.4 Loop ที่ตัวนับเพิ่มแบบ Geometric
for i = 1; i <= n; i = i * 2: print(i)

การนับ: $i$ ผ่านค่า $1, 2, 4, 8, \ldots, 2^k$ จนกว่า $2^k > n$

$2^k \leq n \Rightarrow k \leq \log_2 n$ ดังนั้นจำนวนรอบ $= \lfloor \log_2 n \rfloor + 1$

Complexity: $T(n) = O(\log n)$ — Logarithmic!

// Sequential Search — หา target ใน array a[1..n] sequential_search(a, n, target): for i = 1 to n: if a[i] == target: // การเปรียบเทียบ (basic op) return i // พบ! return -1 // ไม่พบ

11.4

Exponential and Logarithmic Functions: Graphs and Orders

ฟังก์ชัน Exponential และ Logarithmic — คุณสมบัติและลำดับ magnitude

ฟังก์ชัน Exponential

คุณสมบัติ

สำหรับ $a > 0, a \neq 1$ และ $m, n \in \mathbb{R}$:

  • $a^{m+n} = a^m \cdot a^n$
  • $a^{m-n} = a^m / a^n$
  • $(a^m)^n = a^{mn}$
  • $(ab)^n = a^n b^n$
  • $a^0 = 1$, $a^1 = a$
  • ถ้า $a > 1$: $a^x$ เป็น strictly increasing; ถ้า $0 < a < 1$: $a^x$ เป็น strictly decreasing
ตัวอย่าง 11.4.1 ค่า $2^n$ สำหรับ $n$ ต่างๆ
$n$$2^n$ความหมาย
101,024$\approx$ 1 พัน (1K)
201,048,576$\approx$ 1 ล้าน (1M)
301,073,741,824$\approx$ 1 พันล้าน (1G)
324,294,967,296$\approx$ 4G (int unsigned)
64$\approx 1.8 \times 10^{19}$$\approx$ 18E (long long unsigned)

ตรวจสอบ: $2^{10} = 1024 \approx 10^3$ ดังนั้น $2^{10k} \approx 10^{3k}$

ตัวอย่าง 11.4.2 พิสูจน์ว่า $n^k = O(a^n)$ สำหรับ $a > 1$, $k \geq 1$

Polynomial ทุกตัวโตช้ากว่า exponential เสมอ ไม่ว่า degree จะสูงแค่ไหน

ใช้ L'Hôpital's rule (calculus): $\lim_{n\to\infty} \dfrac{n^k}{a^n} = 0$ เพราะ $a^n$ โตเร็วกว่า $n^k$ เสมอ

หมายความว่า: $n^k \leq a^n$ สำหรับ $n$ ใหญ่พอ ดังนั้น $n^k = O(a^n)$ ∎

ฟังก์ชัน Logarithmic

คุณสมบัติ

สำหรับ $a > 0, a \neq 1$ และ $x, y > 0$:

  • $\log_a(xy) = \log_a x + \log_a y$
  • $\log_a(x/y) = \log_a x - \log_a y$
  • $\log_a(x^r) = r \log_a x$
  • $\log_a 1 = 0$, $\log_a a = 1$
  • Change of Base: $\log_a n = \dfrac{\log_b n}{\log_b a}$ สำหรับ $b > 0, b \neq 1$
ตัวอย่าง 11.4.3 $2^{10} = 1024$ และ $\log_2(1024) = 10$

ตรวจสอบ: $\log_2(1024) = \log_2(2^{10}) = 10 \cdot \log_2 2 = 10$ ✓

Change of base: $\log_2(1024) = \dfrac{\ln 1024}{\ln 2} = \dfrac{6.931\ldots}{0.693\ldots} = 10$ ✓

ความหมายใน CS: ต้องใช้ $\lceil \log_2 n \rceil$ bits เพื่อ represent จำนวนเต็ม $n$ ตัว

ตัวอย่าง 11.4.4 ทำไม log ทุก base จึงเป็น $\Theta$ เดียวกัน

จาก Change of Base: $\log_a n = \dfrac{\log_b n}{\log_b a} = \dfrac{1}{\log_b a} \cdot \log_b n$

เนื่องจาก $\dfrac{1}{\log_b a}$ เป็นค่าคงที่ (ไม่ขึ้นกับ $n$)

ดังนั้น $\log_a n = \Theta(\log_b n)$ สำหรับทุก $a, b > 1$

ข้อสรุป: ใน Big-O notation เราสามารถเขียน $O(\log n)$ โดยไม่ต้องระบุ base เพราะ base ต่างกันให้ผลต่างกันแค่ constant factor

ตัวอย่าง 11.4.5 log โตช้ากว่า polynomial ทุกตัว

พิสูจน์: $\log_2 n = O(n^\epsilon)$ สำหรับทุก $\epsilon > 0$

ใช้ substitution $n = 2^m$: $\log_2(2^m) = m$ และ $(2^m)^\epsilon = 2^{\epsilon m}$

ต้องแสดง $m \leq c \cdot 2^{\epsilon m}$ ซึ่งเป็นจริงสำหรับ $m$ ใหญ่พอ (เพราะ exponential โตเร็วกว่า linear)

ตัวเลข: $\log_2(10^9) \approx 30$ ในขณะที่ $(10^9)^{0.01} = 10^{0.09} \approx 1.23$ — เล็กกว่า! แต่สำหรับ $n$ ใหญ่มากพอ $(10^{100})^{0.01} = 10$ ยังน้อยกว่า $\log_2(10^{100}) \approx 332$

ทฤษฎีบท: Order Relations

  • $O(\log n) \subsetneq O(n^\epsilon)$ สำหรับทุก $\epsilon > 0$
  • $O(n^k) \subsetneq O(n^{k+1})$ สำหรับทุก $k \geq 0$
  • $O(n^k) \subsetneq O(a^n)$ สำหรับทุก $k \geq 0$ และ $a > 1$
  • $O(a^n) \subsetneq O(n!)$ สำหรับ $a > 1$

ตั้ง Recurrence

ให้ $T(n)$ = จำนวน comparisons ทั้งหมดสำหรับ input ขนาด $n$

$T(1) = 0$ (base case: 1 element ไม่ต้องทำอะไร)

$T(n) = 2T(n/2) + n$ สำหรับ $n \geq 2$

(แบ่ง 2 ส่วน $\Rightarrow 2T(n/2)$; merge $\Rightarrow n$ comparisons)

แก้ด้วย Unrolling

$T(n) = 2T(n/2) + n$

$\quad = 2[2T(n/4) + n/2] + n = 4T(n/4) + 2n$

$\quad = 4[2T(n/8) + n/4] + 2n = 8T(n/8) + 3n$

$\quad = 2^k T(n/2^k) + kn$

เมื่อ $2^k = n$ (คือ $k = \log_2 n$): $T(n) = n \cdot T(1) + n \log_2 n = 0 + n\log_2 n$

สรุป

$T(n) = n \log_2 n = \Theta(n \log n)$ ∎

Master Theorem (เบื้องต้น)

Master Theorem

ให้ recurrence $T(n) = aT(n/b) + f(n)$ โดย $a \geq 1$, $b > 1$, $f(n) \geq 0$ แล้ว:

  • กรณีที่ 1: ถ้า $f(n) = O(n^c)$ โดย $c < \log_b a$ แล้ว $T(n) = \Theta(n^{\log_b a})$

  • กรณีที่ 3: ถ้า $f(n) = \Omega(n^c)$ โดย $c > \log_b a$ แล้ว $T(n) = \Theta(f(n))$

Analysis of Algorithm Efficiency — สัญลักษณ์ O, Ω, Θ และการวิเคราะห์อัลกอริทึม

11.1 Real Functions

  • $\lfloor x \rfloor$ = จำนวนเต็มที่ใหญ่ที่สุดที่ $\leq x$
  • $\lceil x \rceil$ = จำนวนเต็มที่เล็กที่สุดที่ $\geq x$
  • $\lfloor n/2 \rfloor + \lceil n/2 \rceil = n$
  • Growth: $1 \ll \log n \ll \sqrt{n} \ll n \ll n\log n \ll n^2 \ll 2^n \ll n!$

11.2 O, Ω, Θ

  • $O$: upper bound — $f \leq c\cdot g$ สำหรับ $n \geq n_0$
  • $\Omega$: lower bound — $f \geq c\cdot g$ สำหรับ $n \geq n_0$
  • $\Theta$: tight bound — $c_1 g \leq f \leq c_2 g$
  • $p(n) = \Theta(n^{\deg p})$ สำหรับ polynomial ทุกตัว

11.3 Algorithm Analysis I

  • นับ basic operations เป็น function ของ input size $n$
  • Linear Search: $O(n)$ worst, $O(1)$ best
  • Insertion Sort: $O(n^2)$ worst, $O(n)$ best (sorted)
  • Loop ซ้อน: $O(n^k)$ สำหรับ $k$ ระดับ

11.4 Exp & Log

  • $\log_a n = \log n / \log a$ (change of base)
  • $O(\log_a n) = O(\log_b n)$ (base เปลี่ยนแค่ constant)
  • $n^k = O(a^n)$ สำหรับ $a > 1$ (polynomial โตช้ากว่า exp)
  • $\log n = O(n^\epsilon)$ สำหรับทุก $\epsilon > 0$

11.5 Algorithm Analysis II

  • Binary Search: $O(\log n)$ — แต่ละรอบลด search space ครึ่งหนึ่ง
  • Merge Sort: $T(n) = 2T(n/2) + n \Rightarrow \Theta(n\log n)$
  • Master Theorem: แก้ $T(n) = aT(n/b) + f(n)$
  • Binary Search เร็วกว่า Linear Search $\approx n / \log n$ เท่า
