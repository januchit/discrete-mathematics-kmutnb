12.1

Formal Languages and Regular Expressions

ภาษาทางการและนิพจน์ปกติ

Alphabet, String และ Language

นิยาม: Alphabet, String, Kleene Star

  • Alphabet ($\Sigma$): เซตจำกัดที่ไม่ว่างของสัญลักษณ์ เช่น $\Sigma = \{0, 1\}$ หรือ $\Sigma = \{a, b, c, \ldots, z\}$
  • String (สตริง): ลำดับจำกัดของสัญลักษณ์จาก $\Sigma$ เช่น $w = 01101$ หรือ $w = \text{hello}$
  • Empty string ($\lambda$ หรือ $\varepsilon$): สตริงที่มีความยาวเป็นศูนย์
  • $\Sigma^*$: เซตของสตริงทั้งหมดที่สร้างได้จาก $\Sigma$ รวมถึง $\lambda$ เช่น $\{0,1\}^* = \{\lambda, 0, 1, 00, 01, 10, 11, 000, \ldots\}$
  • Language ($L$): สับเซตใดๆ ของ $\Sigma^*$ กล่าวคือ $L \subseteq \Sigma^*$

ความยาวของสตริง $w$ เขียนว่า $|w|$ เช่น $|01101| = 5$ และ $|\lambda| = 0$ การเชื่อมสตริง (concatenation) $xy$ คือการนำ $x$ ต่อด้วย $y$ เช่น $x = ab$, $y = cd$ ได้ $xy = abcd$

ตัวอย่าง 12.1.1 ตัวอย่าง Languages บน $\{a, b\}$

ให้ $\Sigma = \{a, b\}$ พิจารณา Languages ต่อไปนี้:

  • $L_1 = \{a, ab, abb, abbb, \ldots\} = \{ab^n \mid n \geq 0\}$ — สตริงที่ขึ้นต้นด้วย $a$ ตามด้วย $b$ กี่ตัวก็ได้
  • $L_2 = \{w \in \{a,b\}^* \mid w \text{ มีความยาวคู่}\}$ — สตริงความยาวคู่ทั้งหมด
  • $L_3 = \emptyset$ — ภาษาว่าง ไม่มีสตริงใดเลย
  • $L_4 = \{\lambda\}$ — ภาษาที่มีเฉพาะ empty string
  • $L_5 = \{a,b\}^*$ — ภาษาที่มีทุกสตริงที่เป็นไปได้
ตัวอย่าง 12.1.2 การดำเนินการบน Languages

ให้ $L_1 = \{a, ab\}$ และ $L_2 = \{b, bb\}$ บน $\Sigma = \{a, b\}$

Union: $L_1 \cup L_2 = \{a, ab, b, bb\}$

Concatenation: $L_1 L_2 = \{ab, abb, ab^2, ab^3\} = \{ab, abb, abb, abbb\}$ (เอาสตริงจาก $L_1$ ต่อกับสตริงจาก $L_2$ ทุกคู่)

$L_1 L_2 = \{a \cdot b,\ a \cdot bb,\ ab \cdot b,\ ab \cdot bb\} = \{ab, abb, ab^2, ab^3\}$

Kleene Star: $L_1^* = \{\lambda, a, ab, aa, aab, aba, abab, \ldots\}$ — สตริงทั้งหมดที่เป็น concatenation ของสมาชิกใน $L_1$ กี่ตัวก็ได้ (รวมถึง 0 ตัว = $\lambda$)

การดำเนินการหลักบน Regular Expressions

match สตริงที่เป็น concatenation ของสิ่งที่ match $r$ กี่ครั้งก็ได้ (รวม 0 ครั้ง = $\lambda$)
ลำดับความสำคัญ (Precedence): Kleene star (*) มีความสำคัญสูงสุด รองลงมาคือ Concatenation และ Union (|) มีความสำคัญต่ำสุด เช่น $ab^*|c = (a(b^*))|c$

นิยามอย่างเป็นทางการของ Regular Expression

นิยาม

Regular Expression (RE) บน Alphabet $\Sigma$ นิยามแบบ recursive ดังนี้:

  1. Base cases:
    • $\emptyset$ เป็น RE แทน empty language $\{\}$
    • $\lambda$ (หรือ $\varepsilon$) เป็น RE แทน language $\{\lambda\}$
    • สำหรับแต่ละ $a \in \Sigma$: $a$ เป็น RE แทน language $\{a\}$
  2. Recursive cases: ถ้า $r$ และ $s$ เป็น RE แล้ว
    • $(r)(s)$ เป็น RE แทน concatenation: $L(r) \cdot L(s)$
    • $(r|s)$ เป็น RE แทน union: $L(r) \cup L(s)$
    • $(r^*)$ เป็น RE แทน Kleene star: $L(r)^*$

$L(r)$ หมายถึง language ที่ RE $r$ อธิบาย (describe)

หมายเหตุ: ในทางปฏิบัติมักละวงเล็บออกเมื่อไม่กำกวม เช่น $a^*b^*$ แทน $(a^*)(b^*)$ และ $a|bc$ แทน $a|(bc)$ เพราะ concatenation มีความสำคัญสูงกว่า union

คำอธิบาย: $(0|1)^*$ ตามด้วย $1$ ตามด้วย $(0|1)^*$ ตามด้วย $0$ — หมายความว่าสตริงต้องมีทั้ง 1 และ 0 และ 0 ตัวสุดท้ายต้องมี 1 นำหน้า

Regular Expressions ในการเขียนโปรแกรม

RE ที่ใช้ในโปรแกรม (เช่น Python, JavaScript, grep) เป็นส่วนขยายของ RE ทางทฤษฎี มี shorthand หลายอย่าง:

# ตัวอย่าง Regex ในทางปฏิบัติ [a-z] # = a|b|c|...|z (character class) [0-9] # = 0|1|...|9 \d # = [0-9] (digit) \w # = [a-zA-Z0-9_] (word character) \s # = [ \t\n\r] (whitespace) r{n} # r ซ้ำกัน n ครั้งพอดี r{m,n} # r ซ้ำ m ถึง n ครั้ง r? # = r|λ (optional, 0 หรือ 1 ครั้ง) r+ # = rr* (1 ครั้งขึ้นไป) . # = Σ\{newline} (any character)

หมายเลขโทรศัพท์ไทย: 0XX-XXXXXXX (เบอร์มือถือ 10 หลัก)

0[689]\d-\d{7} # หรือแบบไม่มี dash: 0[689]\d{8}

  • [689] — หลักที่ 2 เป็น 6, 8 หรือ 9

  • - — เครื่องหมาย dash
  • \d{7} — 7 หลักสุดท้าย

อธิบาย:

  • [a-zA-Z0-9._%+-]+ — ชื่อผู้ใช้ (ตัวอักษร, ตัวเลข, หรือ .<>_%+-)

  • [a-zA-Z0-9.-]+ — domain name

12.2

Finite-State Automata

ออโตมาตาแบบจำกัดสถานะ (DFA และ NFA)

นิยาม Deterministic Finite Automaton (DFA)

นิยาม

Deterministic Finite Automaton (DFA) คือ 5-tuple $M = (Q, \Sigma, \delta, q_0, F)$ ที่:

  • $Q$ — เซตจำกัดของ states (สถานะ)
  • $\Sigma$ — alphabet (อักษร)
  • $\delta: Q \times \Sigma \to Q$ — transition function (ฟังก์ชันเปลี่ยนสถานะ)
  • $q_0 \in Q$ — start state / initial state (สถานะเริ่มต้น)
  • $F \subseteq Q$ — เซตของ accept states / final states (สถานะยอมรับ)

Language ที่ DFA รับ

DFA $M$ รับ (accepts) สตริง $w = a_1 a_2 \cdots a_n$ ก็ต่อเมื่อ:

  1. เริ่มที่ $q_0$
  2. อ่านสัญลักษณ์ทีละตัว: $q_0 \xrightarrow{a_1} q_1 \xrightarrow{a_2} q_2 \cdots \xrightarrow{a_n} q_n$
  3. สถานะสุดท้าย $q_n \in F$

Language ของ $M$: $L(M) = \{w \in \Sigma^* \mid M \text{ รับ } w\}$

ถ้า $L(M) = L$ สำหรับ DFA $M$ บางตัว เราเรียก $L$ ว่า Regular Language (ภาษาปกติ)

สัญลักษณ์ใน State Diagram

  • วงกลม: แต่ละ state $q \in Q$
  • วงกลมคู่: accept states ($q \in F$)
  • ลูกศรชี้เข้า: start state $q_0$ (มีลูกศรจากภายนอก)
  • ลูกศรระหว่าง states: $\delta(q, a) = p$ คือ "อ่าน $a$ ที่ state $q$ ไปที่ state $p$"

ตัวอย่าง 1: DFA รับ strings ที่มี "01" เป็น substring

ตัวอย่าง 12.2.1 DFA สำหรับ $L = \{w \in \{0,1\}^* \mid \text{"01" เป็น substring ของ } w\}$

แนวคิด: ต้องการตรวจว่า $w$ มี "01" ปรากฏอยู่หรือไม่ — ใช้ 3 states

States:

  • $q_0$: ยังไม่เห็น "0" ที่อาจเริ่ม "01" (start state)
  • $q_1$: เห็น "0" ล่าสุด รอดูว่าตามด้วย "1" หรือไม่
  • $q_2$: เห็น "01" แล้ว (accept state) — จะอยู่ที่นี่ตลอด
State Diagram: DFA รับ substring "01"

Transition Table:

Stateอ่าน 0อ่าน 1หมายเหตุ
→ q₀q₁q₀Start state
q₁q₁q₂เห็น "0" แล้ว

ทดสอบ: $w = 100101$

$q_0 \xrightarrow{1} q_0 \xrightarrow{0} q_1 \xrightarrow{0} q_1 \xrightarrow{1} q_2 \xrightarrow{0} q_2 \xrightarrow{1} q_2$ — ลงท้ายที่ $q_2 \in F$ ดังนั้น รับ ✓

$q_0 \xrightarrow{0} q_1 \xrightarrow{0} q_1 \xrightarrow{0} q_1 \xrightarrow{0} q_1$ — ลงท้ายที่ $q_1 \notin F$ ดังนั้น ปฏิเสธ ✗

ตัวอย่าง 2: DFA รับ binary strings ที่หารด้วย 3 ลงตัว

ตัวอย่าง 12.2.2 DFA สำหรับ $L = \{w \in \{0,1\}^* \mid \text{ค่า binary ของ } w \equiv 0 \pmod{3}\}$

แนวคิด: เมื่ออ่านบิตทีละตัวจากซ้ายไปขวา ค่าปัจจุบัน mod 3 จะเปลี่ยนตามกฎ: ถ้า current value $= v$ แล้วอ่านบิต $b$ จะได้ $2v + b$

States: $q_0$ (remainder 0), $q_1$ (remainder 1), $q_2$ (remainder 2)

Transitions:

  • จาก $q_r$ อ่าน $b$: ไปที่ $q_{(2r+b) \bmod 3}$
  • $\delta(q_0, 0) = q_0$, $\delta(q_0, 1) = q_1$
  • $\delta(q_1, 0) = q_2$, $\delta(q_1, 1) = q_0$
  • $\delta(q_2, 0) = q_1$, $\delta(q_2, 1) = q_2$

Accept state: $q_0$ (เพราะ remainder 0 = หารด้วย 3 ลงตัว)

หมายเหตุ: empty string $\lambda$ แทนค่า 0 ซึ่ง $0 \equiv 0 \pmod 3$ ดังนั้น $q_0$ เป็นทั้ง start และ accept

Stateอ่าน 0อ่าน 1ความหมาย
→* q₀q₀q₁ค่า mod 3 = 0
q₁q₂q₀ค่า mod 3 = 1
q₂q₁q₂ค่า mod 3 = 2

ทดสอบ: $w = 1001$ (ค่า binary = 9 = 3×3, หารด้วย 3 ลงตัว)

$q_0 \xrightarrow{1} q_1 \xrightarrow{0} q_2 \xrightarrow{0} q_1 \xrightarrow{1} q_0$ — ลงท้ายที่ $q_0 \in F$ ดังนั้น รับ ✓

ทดสอบ: $w = 10$ (ค่า binary = 2 ไม่หารด้วย 3)

$q_0 \xrightarrow{1} q_1 \xrightarrow{0} q_2$ — ลงท้ายที่ $q_2 \notin F$ ดังนั้น ปฏิเสธ ✗

ตัวอย่าง 3: DFA รับ strings ที่ลงท้ายด้วย "ab"

ตัวอย่าง 12.2.3 DFA สำหรับ $L = \{w \in \{a,b\}^* \mid w \text{ ลงท้ายด้วย "ab"}\}$

States:

  • $q_0$: ยังไม่เห็น "a" ที่จะเริ่ม "ab" / หรือเห็นสถานะที่ไม่อยู่ระหว่างรับ "ab"
  • $q_1$: เห็น "a" ล่าสุด (รอ "b")
  • $q_2$: เห็น "ab" ล่าสุด (accept state)
Stateอ่าน aอ่าน bความหมาย
→ q₀q₁q₀สถานะเริ่มต้น
q₁q₁q₂เห็น a ล่าสุด
* q₂q₁q₀เห็น "ab" ล่าสุด

คำอธิบาย transitions:

  • $\delta(q_0, a) = q_1$: เห็น a ครั้งแรกที่อาจเริ่ม "ab"
  • $\delta(q_0, b) = q_0$: b โดยไม่มี a นำหน้า ย้อนกลับ start
  • $\delta(q_1, a) = q_1$: เห็น a ซ้อน, a ใหม่เป็น a ล่าสุด
  • $\delta(q_1, b) = q_2$: ครบ "ab" แล้ว!
  • $\delta(q_2, a) = q_1$: เห็น a ใหม่หลัง "ab" ต้องรอ b อีกครั้ง
  • $\delta(q_2, b) = q_0$: "abb" สุดท้าย ไม่ลงท้ายด้วย "ab"

ทดสอบ: $w = \text{aabab}$

$q_0 \xrightarrow{a} q_1 \xrightarrow{a} q_1 \xrightarrow{b} q_2 \xrightarrow{a} q_1 \xrightarrow{b} q_2$ — ลงท้ายที่ $q_2 \in F$ ดังนั้น รับ ✓

States: $q_e$ (จำนวน a เป็นคู่ — accept), $q_o$ (จำนวน a เป็นคี่)

ทดสอบ $w = \text{aba}$: $q_e \xrightarrow{a} q_o \xrightarrow{b} q_o \xrightarrow{a} q_e$ — รับ ✓ (มี a สองตัว = คู่)

ทดสอบ $w = \text{ab}$: $q_e \xrightarrow{a} q_o \xrightarrow{b} q_o$ — ปฏิเสธ ✗ (มี a หนึ่งตัว = คี่)

Nondeterministic Finite Automaton (NFA)

นิยาม

NFA คือ 5-tuple $N = (Q, \Sigma, \delta, q_0, F)$ ที่เหมือน DFA ยกเว้น:

  • $\delta: Q \times (\Sigma \cup \{\varepsilon\}) \to \mathcal{P}(Q)$ — transition function ส่งคืน เซตของ states (อาจว่าง)
  • NFA อาจมีหลาย transitions จาก state เดียวกันบน input เดียวกัน
  • NFA อาจมี $\varepsilon$-transitions — เปลี่ยน state โดยไม่ต้องอ่าน input

NFA รับ $w$ ก็ต่อเมื่อมี บางเส้นทางการคำนวณที่ลงท้ายที่ accept state

DFA (Deterministic)

  • แต่ละ (state, symbol) มี transition เดียวพอดี
  • ไม่มี $\varepsilon$-transitions
  • ง่ายต่อการ implement
  • อาจต้องการ states มากกว่า NFA
  • $\delta: Q \times \Sigma \to Q$ (ฟังก์ชันปกติ)

NFA (Nondeterministic)

  • แต่ละ (state, symbol) อาจมี 0, 1 หรือหลาย transitions
  • อาจมี $\varepsilon$-transitions
  • มักมี states น้อยกว่า DFA
  • ออกแบบง่ายกว่า DFA
  • $\delta: Q \times (\Sigma \cup \{\varepsilon\}) \to \mathcal{P}(Q)$

ทฤษฎีบท (Rabin-Scott): NFA และ DFA มีประสิทธิภาพเท่ากัน

สำหรับ NFA $N$ ทุกตัว มี DFA $M$ ที่ $L(M) = L(N)$

การพิสูจน์: ใช้ Subset Construction (Power Set Construction) — states ของ DFA $M$ คือสับเซตของ $Q_N$ (states ของ NFA) ดังนั้น $M$ มีได้สูงสุด $2^{|Q_N|}$ states

ตัวอย่าง 12.2.5 NFA และ Subset Construction

Subset Construction จะสร้าง DFA ที่สมมูล — ซึ่งก็คือ DFA ในตัวอย่าง 12.2.3

12.3

Simplifying Finite-State Automata

การทำให้ออโตมาตาเรียบง่าย — Minimization

DFA Minimization Algorithm

DFA ที่รับภาษาเดียวกันอาจมี states จำนวนต่างกัน เราต้องการหา Minimal DFA — DFA ที่มี states น้อยที่สุดสำหรับ language นั้น

นิยาม: States ที่แยกแยะได้ (Distinguishable States)

States $p, q \in Q$ แยกแยะได้ ก็ต่อเมื่อมีสตริง $w \in \Sigma^*$ ที่ทำให้ DFA ยอมรับจาก $p$ แต่ปฏิเสธจาก $q$ (หรือกลับกัน)

กล่าวคือ $\delta^*(p, w) \in F$ และ $\delta^*(q, w) \notin F$ หรือในทางกลับกัน

ถ้า $p$ และ $q$ ไม่แยกแยะได้ เรียกว่า equivalent (สมมูล) — สามารถรวมเป็น state เดียวได้

1
ขั้นตอน 1: ลบ Unreachable States
ลบ states ที่ไม่สามารถไปถึงได้จาก start state $q_0$ ด้วย input ใดๆ ออกไป (ใช้ BFS หรือ DFS จาก $q_0$)
2
ขั้นตอน 2: Table-Filling Algorithm
สร้างตารางคู่ $(p, q)$ สำหรับทุกคู่ states: ทำเครื่องหมาย (marked) ถ้า $p \in F$ และ $q \notin F$ (หรือกลับกัน) จากนั้น propagate: ถ้า $(p,q)$ ยังไม่ marked แต่มีสัญลักษณ์ $a$ ที่ $(\delta(p,a), \delta(q,a))$ ถูก marked แล้ว ให้ mark $(p,q)$ ด้วย วนซ้ำจนไม่มีการเปลี่ยนแปลง
3
ขั้นตอน 3: รวม Equivalent States
คู่ $(p,q)$ ที่ไม่ถูก marked คือ equivalent states — รวมเป็น state เดียว ปรับ transitions ตามไปด้วย
ผลลัพธ์: Minimal DFA ที่ได้มีจำนวน states น้อยที่สุดที่เป็นไปได้ และเป็น unique (ไม่นับชื่อ states) สำหรับ language นั้น

ขั้นที่ 2: Table-Filling

เริ่มต้น: mark ทุกคู่ที่มีหนึ่ง state ใน $F$ และอีกอันไม่ใน $F$

$F = \{q_1, q_4\}$, non-$F = \{q_0, q_2, q_3\}$

  • อ่าน $a$: $\delta(q_0,a)=q_1$, $\delta(q_2,a)=q_4$ — $(q_1,q_4)$ เป็นสองตัวใน $F$ ทั้งคู่ — ไม่ marked
  • อ่าน $b$: $\delta(q_0,b)=q_2$, $\delta(q_2,b)=q_0$ — $(q_0,q_2)$ คือตัวเอง — ยังไม่ marked

พิจารณาคู่ $(q_0, q_3)$:

  • อ่าน $a$: $\delta(q_0,a)=q_1$, $\delta(q_3,a)=q_2$ — $(q_1,q_2)$ — $q_1 \in F, q_2 \notin F$ — marked! ดังนั้น mark $(q_0,q_3)$

พิจารณาคู่ $(q_2, q_3)$:

  • อ่าน $a$: $\delta(q_2,a)=q_4$, $\delta(q_3,a)=q_2$ — $(q_2,q_4)$ — marked! ดังนั้น mark $(q_2,q_3)$

พิจารณาคู่ $(q_1, q_4)$:

  • อ่าน $a$: $\delta(q_1,a)=q_0$, $\delta(q_4,a)=q_3$ — $(q_0,q_3)$ — marked แล้ว! ดังนั้น mark $(q_1,q_4)$

ผล: คู่ที่ไม่ถูก mark = $(q_0, q_2)$ และ $(q_1, q_4)$

ดังนั้น: $q_0 \equiv q_2$ และ $q_1 \equiv q_4$

Myhill-Nerode Theorem

ทฤษฎีบท Myhill-Nerode

Language $L \subseteq \Sigma^*$ เป็น regular ก็ต่อเมื่อ Nerode right congruence $\sim_L$ มี equivalence classes จำนวนจำกัด

นิยาม $\sim_L$: สำหรับ $x, y \in \Sigma^*$ เขียน $x \sim_L y$ ก็ต่อเมื่อ สำหรับทุก $w \in \Sigma^*$: $xw \in L \iff yw \in L$

จำนวน equivalence classes ของ $\sim_L$ เท่ากับจำนวน states ของ minimal DFA สำหรับ $L$

ตัวอย่าง 12.3.2 หา Myhill-Nerode Classes สำหรับ $L = \{w \mid w \text{ ลงท้ายด้วย "ab"}\}$

พิจารณา equivalence classes บน $\{a,b\}^*$:

  • $[q_0]$: สตริง $x$ ที่ไม่ลงท้ายด้วย a หรือ ab — เช่น $\lambda, b, bb, ba, bba$
  • $[q_1]$: สตริง $x$ ที่ลงท้ายด้วย a (แต่ไม่ใช่ "ab") — เช่น $a, aa, ba, baa$
  • $[q_2]$: สตริง $x$ ที่ลงท้ายด้วย "ab" — เช่น $ab, aab, bab$

มี 3 classes ดังนั้น minimal DFA มี 3 states — ซึ่งตรงกับ DFA ในตัวอย่าง 12.2.3

Pumping Lemma สำหรับ Regular Languages

ทฤษฎีบท: Pumping Lemma for Regular Languages

ถ้า $L$ เป็น regular language แล้วมีจำนวนเต็ม $p \geq 1$ (เรียกว่า pumping length) ที่สำหรับทุกสตริง $s \in L$ ที่ $|s| \geq p$ สามารถเขียน $s = xyz$ โดย:

  1. $|xy| \leq p$
  2. $|y| \geq 1$ (กล่าวคือ $y \neq \lambda$)
  3. สำหรับทุก $i \geq 0$: $xy^iz \in L$

กล่าวคือ สามารถ "pump" (ซ้ำส่วน $y$) กี่ครั้งก็ได้แล้วยังอยู่ใน $L$

การใช้ Pumping Lemma แสดงว่า $L$ ไม่ regular: สมมติว่า $L$ regular แล้วใช้ Pumping Lemma นำไปสู่ contradiction — แสดงว่าสมมติฐานผิด ดังนั้น $L$ ไม่ regular (Proof by Contradiction)
ตัวอย่าง 12.3.3 พิสูจน์ว่า $L = \{0^n 1^n \mid n \geq 0\}$ ไม่ Regular

พิสูจน์โดยใช้ Pumping Lemma:

สมมติว่า $L$ เป็น regular และ $p$ คือ pumping length

เลือก $s = 0^p 1^p \in L$ (ความยาว $2p \geq p$)

โดย Pumping Lemma: $s = xyz$ โดย $|xy| \leq p$, $|y| \geq 1$, $xy^iz \in L$ สำหรับทุก $i \geq 0$

เพราะ $|xy| \leq p$: $xy$ อยู่ในส่วน $0^p$ ทั้งหมด ดังนั้น $y = 0^k$ สำหรับบาง $k \geq 1$

พิจารณา $i = 2$: $xy^2z = 0^{p+k} 1^p$ (มี 0 มากกว่า 1 เพราะ $k \geq 1$)

แต่ $0^{p+k} 1^p \notin L$ เพราะจำนวน 0 ไม่เท่ากับจำนวน 1 — contradiction!

ดังนั้น $L$ ไม่ regular ∎

ตัวอย่าง 12.3.4 พิสูจน์ว่า $L = \{ww \mid w \in \{a,b\}^*\}$ ไม่ Regular

สมมติว่า $L$ regular และ $p$ คือ pumping length

เลือก $s = a^p b a^p b \in L$ (โดย $w = a^p b$)

$s = xyz$ โดย $|xy| \leq p \Rightarrow y = a^k$ สำหรับบาง $k \geq 1$ (อยู่ในส่วน $a^p$ แรก)

พิจารณา $i = 0$: $xz = a^{p-k} b a^p b$

ถ้า $xz \in L$ ต้องมีรูป $ww$ คือ $|xz|$ ต้องเป็นเลขคู่และครึ่งแรก = ครึ่งหลัง

แต่ $xz = a^{p-k} b a^p b$ — ส่วนแรกมี $a$ น้อยกว่าส่วนหลัง ($p-k < p$) — ไม่ใช่รูป $ww$ — contradiction ∎

Kleene's Theorem — การเชื่อมโยงทุกอย่างเข้าด้วยกัน

ทฤษฎีบท Kleene (สรุปสูงสุดของบทนี้)

สำหรับ language $L \subseteq \Sigma^*$ สามเงื่อนไขต่อไปนี้สมมูลกัน:

  1. $L$ เป็น Regular Language — มี DFA $M$ ที่ $L(M) = L$
  2. $L$ อธิบายได้ด้วย Regular Expression — มี RE $r$ ที่ $L(r) = L$
  3. $L$ รับได้โดย NFA บางตัว
วงจรการเปลี่ยนระหว่างแบบจำลอง

Thompson Construction: แปลง RE เป็น NFA (with ε-transitions) อย่างเป็นระบบ

Subset Construction: แปลง NFA เป็น DFA สมมูล (อาจมี states มากขึ้น)

State Elimination: แปลง DFA กลับเป็น RE (ลบ states ทีละตัว สร้าง RE สำหรับ transition)

Regular Expressions and Finite-State Automata

12.1 Regular Expressions

  • $\Sigma$: Alphabet, $\Sigma^*$: ทุกสตริง
  • Language $L \subseteq \Sigma^*$
  • RE: $\emptyset, \lambda, a \in \Sigma$
  • $(rs)$ — concatenation
  • $(r|s)$ — union/alternation
  • $(r^*)$ — Kleene star
  • $r^+ = rr^*$, $r? = r|\lambda$

12.2 DFA / NFA

  • DFA $M = (Q, \Sigma, \delta, q_0, F)$
  • $\delta: Q \times \Sigma \to Q$ (deterministic)
  • $L(M)$ = language ที่ DFA รับ
  • NFA: $\delta: Q \times (\Sigma \cup \{\varepsilon\}) \to \mathcal{P}(Q)$
  • NFA $\equiv$ DFA (Rabin-Scott)
  • Subset Construction: NFA → DFA
  • Double circle = accept state

12.3 Simplification

  • Minimization: ลบ unreachable, merge equivalent
  • Table-filling algorithm
  • Myhill-Nerode: classes = states
  • Pumping Lemma: พิสูจน์ไม่ regular
  • Kleene's Theorem: RE ↔ NFA ↔ DFA
  • Regular languages ปิดภายใต้ ∪, ∩, complement
